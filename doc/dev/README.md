## 文档原则

- **精简**：每份文档单次可读完（<500 行），不堆废话
- **决定过的事不再讨论**：核心技术栈已经锁定，出现新信息再开新文档记录"为什么改变了"，不回头改旧文档
- **代码变化优先于文档**：文档只记录设计意图和决策理由，不记录"现在代码里长什么样"（那是代码自己的事）
