# /write — Academic Paper Writing

When the user invokes /write, draft the requested paper section using analysis outputs already on disk. This command handles any paper section and coordinates the full writing workflow. It is distinct from `/method`, which generates only the empirical strategy subsection with equations.

## Step 1: Identify Target

Ask which section to draft (or offer to draft the full paper):

| Section | Primary Skill |
|---------|--------------|
| Introduction | `paper-writing` + `literature-review` |
| Literature Review / Related Work | `literature-review` |
| Data | `paper-writing` + `stats` (Table 1) |
| Empirical Strategy / Methods | `paper-writing` (use `/method` for equations) |
| Results | `paper-writing` + `table` skill for regression tables |
| Robustness | `paper-writing` + `/robustness` output |
| Discussion | `paper-writing` |
| Conclusion | `paper-writing` |
| Full paper | All of the above, sequentially |

## Step 2: Gather Inputs

Read any existing outputs in the working directory:
- Analysis code and results (`analysis_*.py/.R/.do`)
- Regression tables (`tables/*.tex`, `tables/*.csv`)
- Figures (`figures/*.pdf`, `figures/*.png`)
- Notes or outlines the user has already written

Summarize what is available and what is missing before drafting.

## Step 3: Draft

Apply the `paper-writing` skill templates for the requested section:

**Introduction**: Motivate the question, state the contribution, preview findings, and outline the paper. If a literature review is needed, use the `literature-review` skill to identify key references and position the paper.

**Data section**: Describe the data source, sample construction, and key variables. Integrate Table 1 from the `stats` skill. Reference the data cleaning steps from `/data`.

**Results section**: Present the main estimates in prose, referencing coefficient tables generated by the `table` skill. Describe magnitude, direction, and statistical significance. Avoid over-interpreting; use causal language only when identification strategy supports it.

**Discussion / Conclusion**: Interpret findings in context, acknowledge limitations, and state policy or theoretical implications.

## Step 4: Format Tables and Figures

If tables or figures are needed but not yet generated:
- Regression / summary tables → use `table` skill (outputs `.tex` via `estout`/`stargazer`/`modelsummary`)
- Figures → use `figure` skill (AER/QJE standards)

Offer to embed table and figure references (LaTeX `\ref{}`) into the draft.

## Step 5: Save

Write the draft to:
- `writing/[section].md` — Markdown for editing
- `writing/[section].tex` — LaTeX version (on request)

List all bibliography entries needed and offer to generate a `.bib` file with properly formatted BibTeX entries (see `literature-review` skill for citation format guidelines).

## Recommended Workflow

Run these commands in order before `/write`:
1. `/data` — prepare and describe data
2. `/analyze` → `/diagnose` → `/robustness` — run and validate the analysis
3. `/plot` — generate all figures
4. `/method` — pre-generate the empirical strategy subsection with equations

Then call `/write` to assemble the sections into a complete paper draft.
